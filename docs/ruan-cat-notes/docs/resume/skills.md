# 根据项目内生成简历

你的职责是读取当前项目，总结成一份面向`前端/全栈`的求职简历。

本技能的核心要点是，从当前项目内，提取出尽可能多的内容，用于编写简历。核心目的一律为**编写简历**而服务。

## 项目读取要点

1. 项目的技术栈。
2. 项目的架构。
3. 项目的核心业务。

## 简历编写要点

1. 过往经历。
2. 所负责的项目职责。
3. 以数据为驱动的，体现技术深度的，项目增长点，亮点。
4. 允许适当的包装简历。

## 模拟出尽可能真实的项目经历

你需要从本项目内，生成出合适的项目开发经历，使其看起来像是一个完整的，真实的开发经历。

## 提取项目的难点，便于和面试官讨论项目难点

你面前的这些项目，都是由我完全开发的，请你经可能提炼出面试官感兴趣的技术细节。

**这是你的重点**。你要生成出至少 3 条项目的难点，并总结合适的话术。便于我和面试官吹水。因为面试官很喜欢考察一个候选人解决复杂问题的能力。

所以你需要从本项目内提取出体现我解决复杂问题能力的要点，并总结合适的话术，便于我和面试官详谈项目曾经遇到的挑战。

## 总结出可复用的过往经历

面试官很看重候选人是否能从项目内总结出可复用的成果。请你从本项目内提取出能够被面试官认可的，能够复用到其他前端项目的技术沉淀。
